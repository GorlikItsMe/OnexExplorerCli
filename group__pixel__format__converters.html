Pixel format converters module

Contents

  • Reference
    • Functions

Each decode function takes raw NOS pixel bytes (no entry-specific header) and returns 8-bit RGBA bytes in R,G,B,A order, row-major, width*height*4 bytes total. Each encode function is the inverse: RGBA8 in, NOS bytes out.

The input span must contain exactly the bytes the format requires for the given dimensions; otherwise an empty vector is returned.

std::vector<uint8_t> decode_argb555_to_rgba(std::span<const uint8_t> pixels, int width, int height)

ARGB555 -> RGBA8. 1-bit alpha, 5-bit RGB, packed as uint16 LE. 2 bytes/px. Lossy: alpha collapses to 0/255 and channels expand by *8.

std::vector<uint8_t> decode_nstc_to_rgba(std::span<const uint8_t> pixels, int width, int height)

NSTC -> RGBA8. 1 byte palette index into the 16-entry NSTC palette. 1 byte/px. Lossy: only palette colours are representable.

std::vector<uint8_t> decode_bgra8888_interlaced_to_rgba(std::span<const uint8_t> pixels, int width, int height)

BGRA8888 interlaced -> RGBA8. Same packing as BGRA8888 but pixels are laid out in 256-pixel-wide column blocks. 4 bytes/px.

std::vector<uint8_t> decode_barg4444_to_rgba(std::span<const uint8_t> pixels, int width, int height)

BARG4444 -> RGBA8. RGBA, 4 bits per channel, different byte order than GBAR. 2 bytes/px.

std::vector<uint8_t> decode_grayscale_to_rgba(std::span<const uint8_t> pixels, int width, int height)

Grayscale -> RGBA8. 1 luminance byte per pixel, expanded to (g,g,g,g). 1 byte/px. Round-trips only when R=G=B=A.
